## 스코프 (P0 3건 — 이 외 추가 금지)

### ① 국민연금 간이 계산 + 소득 크레바스
- **간이 산식**: 월소득(B값)·가입기간 2입력 → 예상수령액 근사. A값(전체 가입자 평균소득)은 연도 상수로 내장.
  조기수령(-30%)/정상/연기(+36%) 3버튼 비교. **인증 0회·온디바이스.**
- **정확값 우회로(넛지)**: 간이 결과값 바로 옆 "(간이 추정입니다) 공단 조회값 직접 입력 ✏️" 버튼 —
  오차 시비 원천 차단. 직접 입력값이 있으면 간이값보다 항상 우선.
- **크레바스 통합**: 시뮬레이션에서 국민연금 개시 연도부터 연간 목표 인출액에서 수령액 자동 차감 →
  "개시 전 공백기를 연금계좌로 얼마나·어떤 순서로 메꾸는가"가 결과에 드러나게.
  ⚠️ v1.1에서 국민연금은 **현금흐름으로만** 통합 — 국민연금 소득의 종합과세 상호작용은 미반영(보수 가정, 팁 카드 고지). 기존 엔진 v2의 절벽·한도 로직은 사적연금 인정액 기준 그대로.

### ② 건강보험료 반영 (소득 기준 한정)
- 은퇴 후 지역가입자 전환 가정, **소득 기준 보험료만** 추정. 결과에 세금+건보료 합산 표기.
- **Task 1(세법 검증)에서 반드시 확정할 것**: 공적연금 소득 반영률(50%)·사적연금(연금저축/IRP)의
  건보료 부과 여부(현행 미부과로 알려짐 — 사실이면 **"연금계좌 인출은 건보료에 안 잡힌다"가 킬러 메시지**)·
  2024-02 개편(자동차 부과 폐지·재산공제 1억) 이후 최신 요율. TAX_RULES.md에 근거 조항과 함께 기록.
- **디스클레이머 상시 표기**: "재산(부동산 등) 기준분 제외한 소득 기준 추정치".
- **피부양자 박탈 경고 배지**: 연소득 2,000만원 초과 시나리오에 ⚠️ 표시.
- 재산과표 선택 입력은 **P1로 미룸** (이번 스코프 아님).

### ③ 결과 내러티브 "불안 → 숫자"
- 결과 최상단 한 줄 추가: "이 계획이면 자산이 **N세까지** 버팁니다 / 월 **○○만원**을 쓸 수 있습니다"
  (brim 리뷰 검증 감정 가치). 절감액 카드는 그 아래 유지.

## UI (투트랙 — 스텝퍼 반려됨)

1. **홈 폼**: `InputSectionCard` 5번째 "국민연금 (선택)" 접이식 카드 — 기본 접힘, 미입력 시 기존 동작 100% 동일(기존 사용자 마찰 0).
2. **미니 계산기 화면**: 월소득·가입기간 2필드 단독 화면, 홈 상단 진입점. "국민연금 계산기" 검색 유입자가 3탭 내 결과 확인.
3. **auto-fill 퍼널 (필수)**: 미니 결과 → "인출 설계까지 해보기" → 홈 이동 시 예상수령액·개시연령을 5번째 카드에 자동 채움 + **카드 펼침 상태로 진입**. 값은 기존 입력 저장 로직(SharedPreferences)과 동일 패턴으로 유지.
